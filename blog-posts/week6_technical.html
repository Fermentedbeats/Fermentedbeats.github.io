Variables

Date: October 3, 2014

Tags: Dev Bootcamp, DBC, Ruby

As we know, variables are names for stored data. Much like the occupational names in English history, where "Smith" denoted someone who worked as a blacksmith, Ruby also gives clues to where the variable works a.k.a the scope.

Ruby has four main types of variable scopes. Let's dig in:

Local Variables: are typically the first kind you'll come across in Ruby. They are simply a variable name that begins with a lowercase letter or underscore. Local variables are safe: they declare a value and that value can only be accessed from the location of the variable (within a loop or method for example).



Global Variables up the danger level. These variables are available to everything in the program which means they can be changed anywhere, any time. Global variables start with a $.



Instance Variables: these variables are unique to one class. They make a variable available throughout the class, incuding to the methods within that class. Instance variables start with a @. You'll notice in the example that each method call returns an answer specific to that Class.





Class Variables: much like instance variables, class variables are contained to a class, but also sub-classes. The difference is in the inheritance. Class variables have a single value, that once changed, is changed across the whole reach of the class. Unlike our previous example, because the Learning class is a sub-class of Fluency, the value changes permanently, yielding a new value for @@language, set by the sub-class.
