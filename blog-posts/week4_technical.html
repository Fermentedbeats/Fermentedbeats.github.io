Ruby Road.map


Date: September 18, 2014

Ruby's enumerable methods are fast becoming my go-tos. Their magic is found in the significant ability to reduce the word vomit ever-present in newbie code. Today we check out the classic map method:

  • In the dawn of learning Ruby as a complete newcomer to programming, there's an early understanding of needing to "loop." Looping gets something done a bunch of times so you don't go blind writing the same command a thousand times. However, looping without an understanding of enumerables can lead to some weird practices.
  • "While," "times," "until," and "for" loops cover a lot of needs, but what if you don't have a true/false condition or know the number of times something will happen? When I look back at some of my earliest loops for codeacademy and other tutorials I see a lot of "while true" statements paired with later "now it's false statements," constructed only to force a loop. Similarly I found myself liberally jamming in empty arrays to hold my data.
  • Enumerables, as I've learned this week, circumvent these awkward artifical loops. The map method will apply the given code block to each element in your array, returning a new array, like so:

    •       array.map { |x| do_something_to_each_element_of_the_array } => new_array_with_results

  • In practice, the map method iterates for you. No loop construct? No problem.

    •       def good_investment(array_of_customer_investments)
             plus_interest = array_of_customer_investments.map {|x| plus_interest << (x * 1.08) }
          end
