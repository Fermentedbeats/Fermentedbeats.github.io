What is Rails?


Date: October 17, 2014

R ails is a Ruby gem. A powerful one. Rails is additional code, that on top of Ruby, can allow creation and support of dynamic and often complex websites. Rails combines the underlying Ruby language with HTML, CSS, and Javascript to run on a web server where it is ultimately displayed in a web browser. The Rails side of this package is considered the back end.

The rise in poopularity of Rails is the idea of "the Rails way" a set of guidelines and best practices rigidly followed, which means less fewer decisions need to be made from the developer and much of the supporting software has already been built. This also enforces similar practices across projects, easier troubleshooting, and reusability of code. Because Rails handles the database communication, certain updates, and has a lot of available plugins to easily install features developers can place more focus on making 'the cool stuff.' And fast. Rails has vast open-source libraries available to quickly launch a site. Because so much code is readily available developers can make well-structured programs with less effort.
