SQL Injection

Date: October 8, 2014

Tags: Dev Bootcamp, DBC, SQL

As we delve into SQL this week, we learn about SQL Injections; a well-known way of attacking a website with potential malicious action like stealing user info: credit card numbers, social security numbers, passwords, etc.

Typically your SQL hacker would manipulate a url or web form to mine data or wreak havoc. By accessing the database, they can send commands to return what should be hidden data. One such example would be concatenating the data.

This places vigilence on the shoulders of programmers - but luckily some basic measures go a long way toward preventing attacks. List of safeguards for newbies:

  • Setting parameters for user data to match against the values in existing fields can guard against concatenation.
  • Also validating data by content, size and/or type can guard against data scrapes.
  • Setting user permissions keeps the data open to hacking to a minumum.
  • Failing to disclosure the error statements also leaves less crumbs for a hacker to follow.
